Lab 3 - Adding a WAF policy to an application¶

This lab will be broken down to 3 parts.

  1. We will apply an already existing WAF policy to an HTTPS virtual server.
  2. We will apply an policy that is stored in a source repository to an HTTPS virtual server.
  3. We will update the policy to be in blocking mode.
  4. We will delete our application configurations.

Already Existing WAF policy deployment¶

  1. If you already have a WAF policy on the BIG-IP that you would like to apply to your application then it is easy enough to just reference it in the AS3 declaration in the policyWAF area.

Delete the Applications deployed¶

  1. This time we will delete all the AS3 declarations that are currently deployed.
  2. In Postman open the Getting Started section.
  3. Select the BIG-IP: Delete ALL declarations declaration, and look at the Body of the declaration.
  4. In this instance we are setting the Action to remove. Also notice that we set a syncToGroup option in order to cause the HA pair to sync configurations.
